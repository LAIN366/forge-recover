# 实验协议

## 实验组

1. `no-recovery`：异常后直接失败。
2. `fixed-rule`：固定规则重试。
3. `diagnosis-only`：诊断但不主动探测。
4. `active_case`：主动探测 + 案例恢复。
5. `full`：主动诊断 + RAG/LLM 候选 + 约束检查 + 恢复验证。

## 故障集合

- 抓取滑落
- 未抓住
- 目标位姿移动
- 接触传感器噪声
- 路径执行期间出现的动态障碍
- IK 不可达与规划失败
- 后续扩展：视觉遮挡与多故障组合

每类故障至少设置三个严重程度、十个随机种子。正式论文实验建议每个组合不少于
30 个回合。

## 指标

- 异常检测 Precision、Recall、F1
- 故障分类准确率
- 从故障注入到首次诊断的检测延迟
- 主动探测次数、耗时和置信度变化
- 恢复成功率与最终任务成功率
- 平均恢复次数、恢复时长和动作数量
- 碰撞率、安全终止率和二次异常率
- 规划耗时、轨迹长度与关节平滑度

## 可复现要求

- 每回合记录随机种子、故障类型和严重程度。
- 正常组和故障组使用相同初始场景分布。
- 阈值调优数据与最终测试数据隔离。
- 失败回合不得删除，必须保留 JSONL 和控制台日志。
- 任何阈值变化需通过 Git 提交和新批次 ID 追踪。
- 使用 `export_diagnosis_dataset.py` 导出训练数据时，按回合而不是按帧划分
  训练、验证和测试集，避免同一轨迹泄漏到不同数据子集。
